# A2 experiments — run, plot, analyze, replay

Everything below runs with the `cs234a2` conda env:
`/opt/anaconda3/envs/cs234a2/bin/python` (the `base` env has no `gym`).

## The grid

3 games × 3 methods × 3 seeds = **27 runs**.

- **games:** `cartpole`, `pendulum`, `cheetah`
- **methods:** `baseline` (PG + value baseline), `no_baseline` (PG, no baseline), `ppo`
- **seeds:** `1, 2, 3`

## 1. Run everything (parallel) — then auto-plots + analysis

```bash
cd /Users/danielnurieli/RL/CS234-Reinforcement-Learning/A2_Policy_Gradient_Methods/code
OMP_NUM_THREADS=1 /opt/anaconda3/envs/cs234a2/bin/python run_experiments.py --seeds 1,2,3 --jobs 6
```

- Runs all 27 jobs, **6 at a time**, each pinned to 1 CPU thread. The work is bound
  by serial PyBullet env-stepping (~900 steps/s/core), not matrix math, so 6
  single-thread processes ≈ 6× throughput on this 6-core machine.
- **ETA ~1.5–2 h** (cheetah dominates; it's scheduled first). Sequential would be ~7–9 h.
- When done it automatically calls `plot.py` (per game) and `analyze.py`.
- Runs in the foreground; to background it, append `&` or launch under `nohup`.

Subsets:
```bash
# just the fast games
... run_experiments.py --envs cartpole,pendulum --seeds 1,2,3 --jobs 6
# cheetah on its own later
... run_experiments.py --envs cheetah --seeds 1,2,3 --jobs 6
```

## 2. Outputs (per run)

`results/<GymEnvName>-<method>-seed=<s>/`:
- `scores.npy` — per-iteration average reward (used by plots/analysis)
- `scores.png` — that single run's curve
- `log.txt` — training log
- `model.weights/policy.pt` (+ `baseline.pt`) — **saved trained weights** (for replay)
- per-job stdout in `results/_runlogs/<tag>.log`

## 3. Plots (comparison, mean ± SEM across seeds)

Auto-generated by step 1, or manually:
```bash
python plot.py -d results --env-name cartpole --seeds 1,2,3   # -> results/results-cartpole.png
python plot.py -d results --env-name pendulum --seeds 1,2,3
python plot.py -d results --env-name cheetah  --seeds 1,2,3
```
Each figure overlays the three methods (Baseline / No baseline / PPO).

## 4. Analysis table

```bash
python analyze.py --seeds 1,2,3        # -> prints + writes results/analysis.txt
```
Shows final reward (mean of last 20 iters) per game × method, averaged over seeds ± std.

## 5. Replay a saved model (watch it play)

Print rewards only (headless):
```bash
python play.py --env-name cheetah --ppo --seed 1 --episodes 5
```
Watch it live from the side, real-time — **run in your own terminal** (a GUI window
needs your display; it can't open from a background/agent shell):
```bash
python play.py --env-name cheetah --ppo --seed 1 --render --render-yaw 180 --render-fps 60
python play.py --env-name cartpole --ppo --seed 1 --render --render-yaw 0 --render-fps 60
```
Match the flags to the run you want to load: `--ppo`, or `--baseline` / `--no-baseline`.

## Camera notes (for --render)

- **Side view:** `--render-yaw 0` (cartpole) or `--render-yaw 180` (cheetah). `90`/`270` are end-on.
- Speed: `--render-fps 60` ≈ real time; `0` = as fast as possible.
- Tune framing with `--render-pitch` (tilt) and `--render-dist` (zoom).
